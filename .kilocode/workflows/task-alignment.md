# 🔍 开发任务对齐检查流程

> 在开始每个开发任务前，务必执行此检查流程

---

## 📌 使用时机

- 开始新的开发任务时
- 中断后恢复开发时
- 实现任何功能前

---

## ✅ 检查步骤

### 步骤1: 查看当前进度

使用命令 `/status` 查看当前进度仪表板

或直接查看任务清单:
```bash
# 查看任务清单 (在 brain 目录下)
cat ~/.gemini/antigravity/brain/*/task.md
```

### 步骤2: 确定下一个任务

在 task.md 中找到:
- `[ ]` 待开始的任务
- `[/]` 进行中的任务

### 步骤3: 定位设计文档章节

每个任务都标注了设计文档章节引用，例如：
- `§3.1.2` - 欢迎页模块
- `§3.2.3` - 文章管理模块

搜索设计文档:
```bash
grep -n "§3.1.2\|3.1.2" /Users/liangfeng/evn/projects/AetherBlog/系统需求企划书及详细设计.md | head -5
```

### 步骤4: 阅读设计规格

查看对应章节内容（根据搜索结果调整行号）:
```
view_file 系统需求企划书及详细设计.md StartLine:XXXX EndLine:YYYY
```

### 步骤5: 输出理解报告

编码前必须确认:
- [ ] 已识别相关设计章节
- [ ] 已提取技术要求
- [ ] 已理解数据结构
- [ ] 已明确 API 接口
- [ ] 已识别依赖关系

### 步骤6: 检查现有实现

查看是否已有相关代码:
```bash
find apps/ packages/ -name "*.tsx" -o -name "*.ts" -o -name "*.java" | \
  xargs grep -l "组件名称" 2>/dev/null | head -10
```

### 步骤7: 更新进度

完成任务后更新:
1. task.md 中 `[ ]` → `[x]`
2. 使用 `/status` 确认进度
