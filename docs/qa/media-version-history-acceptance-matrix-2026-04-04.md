# 媒体版本管理验收测试矩阵
> 创建日期：2026-04-04

## 测试范围
媒体文件版本历史功能（查看、恢复、删除历史版本）

核心文件：
- `apps/admin/src/pages/media/components/VersionHistory.tsx` - 版本历史组件
- `apps/admin/src/services/versionService.ts` - 版本管理 API 调用
- `apps/server-go/internal/handler/version_handler.go` - 版本管理处理器
- `apps/server-go/internal/service/version_service.go` - 版本业务逻辑
- `apps/server-go/internal/repository/version_repo.go` - 版本数据访问
- `apps/server-go/internal/model/media_version.go` - 版本数据模型

## 2. 版本恢复测试
| 用例 | 操作 | 预期结果 | 状态 |
|------|------|---------|------|
| 选中历史版本 | 点击非最新版本的版本卡片 | 卡片展开，显示"恢复此版本"和"删除"按钮 | - |
| 执行恢复 | 点击"恢复此版本" → 确认弹窗 | 当前文件内容回退至该版本，原当前版本被保存为新版本 | - |
| 恢复后版本列表更新 | 恢复操作完成后 | 版本列表刷新，新版本出现在列表顶部，标记为"当前版本" | - |
| 恢复后媒体列表更新 | 恢复完成后回到媒体列表 | 文件的缩略图/大小等信息已更新 | - |
| 恢复最新版本不可操作 | 查看当前版本 | 最新版本不显示"恢复此版本"按钮 | - |
| 取消恢复 | 点击"恢复此版本" → 取消确认弹窗 | 取消操作，版本历史不变 | - |
| 恢复中防重复提交 | 恢复执行中再次点击 | "恢复此版本"按钮禁用，不触发多次请求 | - |

## 7. API 接口对应
| 操作 | 接口 |
|------|------|
| 获取版本历史列表 | `GET /v1/admin/media/:id/versions` |
| 恢复到指定版本 | `POST /v1/admin/media/:id/versions/:versionNumber/restore` |
| 删除历史版本 | `DELETE /v1/admin/media/versions/:versionId` |

## 8. 阻断级缺陷定义（Blocker）
- 恢复版本后文件内容未实际变更。
- 恢复版本未保存原当前版本（即恢复操作导致版本丢失）。
- 版本删除后文件内容被意外影响。
- 版本列表显示信息与实际不符（大小/时间错误）。
