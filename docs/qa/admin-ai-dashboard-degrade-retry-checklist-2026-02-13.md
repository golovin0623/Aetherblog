# Admin AI 看板降级与重试体验检查（2026-02-13）

## 1. 目标
- AI 看板失败时不再静默使用空 Mock 数据。
- 显示明确降级态（状态徽标 + 错误信息 + 诊断类别）。
- 支持一键重试，并展示最近一次成功时间。
- 日志面板可区分正常 / 暂无日志 / 读取失败。

## 2. 关键行为
- AI 看板：
  - 成功请求后状态为“运行正常”，并更新“最近成功时间”；
  - 非 200 或请求异常后状态为“降级中”，保留最近成功数据；
  - 点击“重试”可立即重新请求，恢复后自动回正。
- 实时日志：
  - 状态徽标显示 `运行正常 / 暂无日志 / 降级中`；
  - 顶栏显示最近成功时间；
  - 提供“立即重试”按钮。

## 3. 手工回归
1. 正常场景：打开 Dashboard，确认 AI 状态为“运行正常”，日志状态为“运行正常”。
2. API 非 200：模拟 `/v1/admin/stats/ai-dashboard` 返回失败，确认显示“降级中”且出现错误提示。
3. 网络超时：人为断网或注入超时，确认“降级中 + 重试按钮”可见。
4. 恢复重连：恢复服务后点击“重试”，状态自动恢复为“运行正常”。
5. 日志空数据：日志文件为空/不存在时确认状态为“暂无日志”并显示诊断信息。
