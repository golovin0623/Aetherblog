# 🤖 代理行为准则 (Agent Behavior Standards)

## 1. 全权负责原则 (Full Ownership Principle)
**禁止将运维负担转嫁给用户。**
- **❌ 错误行为**: 要求用户手动重启服务、手动编译、手动清除缓存。
- **✅ 正确行为**: 
  - 遇到需要重启的场景，必须自动调用 `./start.sh` 或 `docker restart`。
  - 遇到环境问题，必须尝试自动修复。
  - 只有在 AI 权限不足（如需要 sudo 密码且未提供）时，才请求用户协助。

## 2. 交付标准 (Definition of Done)
**交付即验证。**
- 在通知用户之前，必须确保环境已就绪。
- "请验证" 的前提是服务已经成功重启并运行。

## 3.禁止未经用户确认的擅自git 提交、更新、删除、推送操作，查询类的则可自行执行

## 4. 移动端 UI 调试原则
**移动端样式修改必须支持真机验证。**
- 涉及移动端 UI 调整时，提醒用户可通过局域网 IP 在真实设备上预览（`/mobile-debug` 工作流）。
- 移动端断点统一使用 `useMediaQuery('(max-width: 768px)')`，不得引入非标准断点。
- 底部弹出面板使用 Bottom Sheet 模式（`max-h-[66vh]`），不得使用全屏覆盖。
- 修改移动端样式时，不得破坏桌面端已有布局。
